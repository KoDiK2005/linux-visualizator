# Linux Visualizator

Виджет для Linux, который красиво в реальном времени показывает, как работает компьютер:
загрузка CPU по ядрам, использование RAM/Swap, сетевой трафик, заполненность дисков и
скорость чтения/записи (I/O).

Есть две реализации:

- **Python/Qt-приложение** (`core/`, `ui/`) — самостоятельный десктопный виджет на PySide6,
  работает в любом Linux-окружении, запускается как обычный процесс.
- **Нативный Cinnamon-десклет** (`cinnamon-desklet/`) — для Linux Mint/Cinnamon: написан на
  JS/GJS под Cinnamon Desklet API, исполняется внутри самого Cinnamon (без Python/venv),
  добавляется через штатное меню «Десклеты», двигается и настраивается стандартными
  средствами Cinnamon. См. [cinnamon-desklet/README.md](cinnamon-desklet/README.md).

Обе реализации независимо парсят одни и те же источники (`/proc/stat`, `/proc/meminfo`,
`/proc/net/dev`, `/proc/diskstats`) и рисуют один и тот же набор панелей — просто на разных
стеках, поскольку Cinnamon-десклеты физически не могут быть Python/Qt-процессом: их
выполняет сам Cinnamon через GJS.

## Идея (Python/Qt-версия)

Проект разделён на два независимых слоя:

- **core** — сбор и агрегация системных метрик (не зависит от UI, читает `/proc` и использует `psutil`).
- **ui** — десклет на PySide6 (Qt): прозрачное окно без рамки, которое рисует живые индикаторы
  поверх/под рабочим столом.

Такое разделение позволяет в будущем добавить другой UI (например, веб-дашборд) без переписывания
сборщиков данных.

## Структура

```
core/
  collectors/   # cpu.py, memory.py, network.py, disk.py — парсинг /proc и psutil
  models.py     # датаклассы-снапшоты метрик
  sampler.py    # периодический опрос + история для графиков
  config.py     # интервалы опроса, список интерфейсов, пороги
ui/
  widget/
    main_window.py       # frameless always-on-desktop окно, владеет sampler'ом и таймерами
    animation.py         # плавное сглаживание значений между тиками
    app_settings.py       # датакласс настроек + сохранение/загрузка через QSettings
    settings_dialog.py    # диалог настроек (интервал опроса, видимость панелей)
    panels/               # cpu_panel.py, mem_panel.py, net_panel.py,
                           # disk_usage_panel.py, disk_io_panel.py, sparkline_panel.py (общий код)
    theme.py              # цвета, прозрачность, шрифты
  app.py                  # точка входа
scripts/
  install_autostart.sh  # регистрирует десклет в автозапуске (~/.config/autostart)
  uninstall_autostart.sh
tests/
cinnamon-desklet/
  linux-visualizator@KoDiK2005/  # нативный Cinnamon-десклет (JS/GJS), см. его README
  install.sh / uninstall.sh
```

## Статус

Все этапы плана ниже выполнены: рабочий десклет с живой визуализацией, плавными
переходами значений, запоминанием позиции на экране, автозапуском и мини-GUI настроек.

## План работы

0. Скелет репозитория, зависимости, CI — **готово**.
1. Core collectors: CPU, RAM, Network + юнит-тесты парсинга `/proc` — **готово**.
2. Sampler: кольцевой буфер истории для спарклайнов, конфигурируемый интервал опроса — **готово**.
3. Базовый UI-каркас: прозрачное frameless-окно, перетаскивание, сохранение позиции — **готово**.
4. Панели визуализации: кольца загрузки CPU по ядрам, полоса RAM/Swap, спарклайн сети — **готово**.
5. Полировка: анимации значений, автозапуск, консольная точка входа — **готово**.
6. Мини-GUI настроек: правый клик по виджету → интервал опроса, видимость панелей — **готово**.

## Запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -e .
.venv/bin/python -m ui.app
```

На Debian/Ubuntu для работы Qt может понадобиться системный пакет `libxcb-cursor0`
(`sudo apt install libxcb-cursor0`).

Окно перетаскивается мышью — позиция сохраняется в
`~/.config/linux-visualizator/desklet.ini` и восстанавливается при следующем запуске.

## Автозапуск

```bash
./scripts/install_autostart.sh    # включить автозапуск при входе в систему
./scripts/uninstall_autostart.sh  # отключить
```

Скрипт создаёт `~/.config/autostart/linux-visualizator.desktop`, который подхватывается
GNOME, KDE, XFCE, Cinnamon и другими окружениями, поддерживающими XDG autostart.

## Настройки

Правый клик по виджету открывает меню: «Настройки…» и «Выход». Диалог настроек разделён на
три вкладки:

- **Основное** — интервал опроса (200 мс – 5 с), видимость панелей CPU/RAM/Сеть/Диски.
- **Внешний вид** — пороги жёлтого/красного цвета (по умолчанию 70%/90%), масштаб всего
  интерфейса (75–150%, влияет на размер колец/полос/шрифтов), порядок панелей (например,
  `mem,cpu,disk,net`).
- **Данные** — фильтр разделов диска (подстроки точек монтирования через запятую, пусто —
  показывать все), единицы измерения сети (байты KB/s или биты Kb/s), фильтр сетевых
  интерфейсов (через запятую, пусто — суммировать все).

Настройки сохраняются в `~/.config/linux-visualizator/desklet.ini` и применяются мгновенно,
без перезапуска.

Панель дисков показывает полосу заполненности для каждого локального раздела крупнее 1 ГБ
(мелкие вроде `/boot/efi` отсекаются, если не переопределено фильтром) и спарклайн скорости
чтения (R) / записи (W).

Панель CPU показывает над кольцами общую загрузку, среднюю частоту ядер и температуру (если
датчик найден) — например, «7% 2.7 GHz 48°C». Кольца CPU, полоса RAM и полосы разделов диска
подсвечиваются жёлтым/красным при превышении настроенных порогов (тот же механизм, что и в
Cinnamon-десклете). Полоса RAM также показывает абсолютные значения в GB, а SWAP скрывается
автоматически, если своп не настроен.

## CI

GitHub Actions (`.github/workflows/ci.yml`) на каждый push/PR в `main` прогоняет `ruff check`
и `pytest` для Python-кода, плюс валидацию JSON и синтаксиса `desklet.js` для Cinnamon-десклета.

## Требования

- Python 3.11+
- Linux (используется `/proc`)
- PySide6, psutil
