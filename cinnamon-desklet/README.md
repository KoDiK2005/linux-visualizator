# Linux Visualizator — Cinnamon Desklet

Нативный десклет для Linux Mint (Cinnamon): живая визуализация CPU по ядрам, RAM/Swap,
сети и дисков (I/O), без Python/venv — весь код выполняет сам Cinnamon через GJS.

Это независимая реализация той же идеи, что и Python/Qt-приложение из корня репозитория
(см. [../README.md](../README.md)): те же источники данных (`/proc/stat`, `/proc/meminfo`,
`/proc/net/dev`, `/proc/diskstats`) и тот же набор панелей, но на другом стеке — Cinnamon
Desklet API устроен так, что десклеты обязаны быть JS/GJS-скриптами, исполняемыми внутри
процесса `cinnamon`.

## Структура

```
linux-visualizator@KoDiK2005/
  metadata.json          # UUID, имя, описание десклета
  desklet.js              # сбор метрик из /proc + отрисовка через Cairo/Clutter.Canvas
  settings-schema.json    # интервал опроса, видимость панелей (интегрируется в Desklet Settings)
install.sh / uninstall.sh
```

## Установка

```bash
./install.sh
```

Скрипт создаёт симлинк `~/.local/share/cinnamon/desklets/linux-visualizator@KoDiK2005` на
папку в репозитории — изменения в исходниках сразу видны Cinnamon без переустановки.

Дальше добавьте десклет на рабочий стол:

1. Правый клик по рабочему столу → «Десклеты»
2. Найдите «Linux Visualizator» в списке слева и нажмите «+»

Удалить:

```bash
./uninstall.sh
```

(и уберите десклет с рабочего стола через правый клик по нему → «Убрать этот десклет»).

## Настройки

Правый клик по десклету → «Настройки десклета» (доступно благодаря `settings-schema.json`):

- **Интервал опроса** — 200–5000 мс
- **Непрозрачность фона** — 40–255
- **Показывать CPU / RAM-Swap / Сеть / Диски** — независимые переключатели
- **Вид отображения CPU** — кольца по ядрам / полосы по ядрам / график общей загрузки
- **Показывать частоту CPU** — средняя частота ядер из `/proc/cpuinfo`, выводится в
  заголовке секции («3.4 GHz»)
- **Показывать температуру CPU** — если датчик найден в `/sys/class/hwmon` (Intel "Package
  id 0" или AMD "Tctl"/"Tdie"); если нет — просто не показывается
- **Сетевые интерфейсы** — ограничить график сети списком через запятую (например,
  `wlan0,enp3s0`) вместо суммы всех, кроме `lo`
- **Единицы измерения сети** — байты (KB/s) или биты (Kb/s)
- **Разделы дисков** — подстроки точек монтирования через запятую (например, `/, /home`),
  пусто — показывать все разделы крупнее 1 ГБ
- **Масштаб интерфейса** — 75–150%, пропорционально меняет размер колец/полос/шрифтов
- **Порядок секций** — через запятую (`cpu,mem,net,disk`), пропущенные секции добавляются
  в конец
- **Жёлтый / красный порог** — проценты, при превышении которых индикаторы окрашиваются
  (по умолчанию 70%/90%)
- **Цвет CPU / RAM / сети / дисков** — акцентный цвет каждой категории через стандартный
  Cinnamon colorchooser; вторая линия спарклайна (отправлено/запись) рисуется автоматически
  осветлённым оттенком этого же цвета
- **Команда по клику** — что запускать по левому клику на десклете (по умолчанию
  `gnome-system-monitor`); пусто — отключить

Позиция и размер десклета управляются самим Cinnamon (перетаскивание в режиме
редактирования десклетов), отдельного кода для drag/drop не требуется. Клик мышью (без
перетаскивания) обрабатывается через штатный хук `on_desklet_clicked` и не конфликтует с
перемещением десклета.

## Диски

Заполненность разделов получается через `df` (исключены virtual-ФС: tmpfs, overlay,
squashfs и т.п.; разделы меньше 1 ГБ отсекаются). Скорость чтения/записи — из
`/proc/diskstats`, суммируется по физическим блочным устройствам (`sdX`, `nvmeXnY`,
`mmcblkX`), без учёта отдельных партиций, чтобы не задваивать байты.

## Внешний вид

- Заголовки секций с цветным маркером («ПРОЦЕССОР», «ПАМЯТЬ», «СЕТЬ», «ДИСКИ») и тонкие
  разделители между ними.
- Градиентная заливка полос и заливка под линиями спарклайнов.
- Цветовая индикация нагрузки: кольца CPU, полоса RAM и полосы разделов диска подсвечиваются
  жёлтым/красным при превышении настраиваемых порогов (по умолчанию 70%/90%).
- Плавная анимация значений между тиками опроса (~30 кадров/сек, как в Python-версии), а не
  скачки чисел.
- Для строк со стрелками (↓/↑) используется шрифт DejaVu Sans — Cairo toy-text API не делает
  автоматический fallback шрифта по глифу, и без этого стрелки могли отрисовываться как «тофу».

## Известные исправленные баги

- **Паразитные линии между кольцами CPU.** Cairo не сбрасывает «текущую точку» между
  операциями — после `showText()` предыдущего кольца следующий `arc()` неявно проводил
  линию от старой точки до начала новой дуги. Исправлено добавлением `ctx.newSubPath()`
  перед каждой дугой (и перед точкой-маркером в заголовке секции, по той же причине).

## Проверено

Десклет протестирован вживую на Linux Mint 22.3 (Cinnamon): загружается без ошибок в
JS-логе (`~/.xsession-errors`), корректно отрисовывает кольца CPU (без паразитных линий),
полосы RAM/Swap/дисков и спарклайны сети и I/O; все три вида CPU и все настройки (фильтр
интерфейса, непрозрачность фона) проверены переключением вживую. Изменения кода
проверялись через `cinnamon --replace` (перезапуск оболочки без разлогинивания) — иначе
Cinnamon кэширует уже загруженный JS-модуль десклета и не видит правки в файле.

Клик-по-десклету (запуск `gnome-system-monitor`) проверен только по коду и через
официальный API-хук `on_desklet_clicked` — интерактивный клик мышью не тестировался
автоматически (нет доступного инструмента для эмуляции клика на X11-рабочем столе, только
браузерная автоматизация).
